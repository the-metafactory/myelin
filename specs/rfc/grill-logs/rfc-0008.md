## The three Andreas calls (2026-07-15)

### D1 (§6.1) — Capability-id grammar → **converge-widen: dotted-compound, kebab-strict segments**
`capability-id = segment *("." segment)`, each segment RFC-0001 kebab-strict. Matches the RATIFIED tasks subject shape 1:1 (`tasks.{capability}.{subcapability}` — the id's dots project onto subject segment boundaries; no bridge artifact). Underscores are determined by ratified siblings, not style: segments project into subjects, kebab-strict forbids `_` → `federated.subject_dispatch` migrates (`subject-dispatch`) at flag-day R; live `code-review.typescript`, `deploy.k8s`, `dev.implement` already conform. myelin's single-segment `capability-tag` remains the SUBJECT-segment terminal (RFC-0002's); the compound id is the IDENTIFIER whose segments project into it. Converge-narrow (breaks more live traffic, loses the cap/subcap structure) and bridge (third artifact forever) REJECTED.

### D2 (§6.2) — Canonical discovery wire → **presence wire canonical; F-11 retires at R**
The push-model presence protocol (`agent.online` initial set + `agent.capabilities-changed` deltas, on `{scope}.{principal}.{stack}.agent.{action}`, signed 0004 envelopes, TTL liveness FSM — ADR-0007/G-1114) is THE normative discovery wire. Verified: cortex consumes ZERO of F-11 (no references to `registerCapabilities`/`CapabilityAdvertisement`/`SignedCapabilityRegistration`/`verifyCapabilityRegistration` on main). F-11's pull API + the `AGENT_CAPABILITIES` KV contract (§5) RETIRE at flag-day R — §5 becomes historical/informative. F-11's verification semantics fold into the presence path as the D5 trust-boundary rule. §7 (presence wire, currently informative) INVERTS to the normative wire spec. **Consequence (verified in source):** the live presence payloads (`AgentOnlinePayloadSchema`/`AgentCapabilitiesChangedPayloadSchema`, cortex `src/bus/agent-network/envelopes.ts:182,232`) carry identity/scope/capabilities + timestamp ONLY — the F-11-exclusive members (`sovereignty`, `load`, `maxConcurrent`, `updatedAt`) had zero consumers and RETIRE WITH the F-11 wire (recorded historically; a future `Updates:` reintroduces them when a consumer exists). The full-set-not-diff semantics of `capabilities-changed` (converge after a missed delta) and the ADR-0005 no-session-interior rule become normative. D3's equality match applies wherever a declared mode meets a required mode (dispatch admission), independent of discovery carriage. Codify-the-wire-as-it-is: spec the wire everything uses, retire the wire nothing uses.

### D3 (§6.5) — `sovereignty_required` matching → **equality-matched v1; ordering RESERVED**
`match(required, declared) := required == declared`. The partial order the field name implies is explicitly RESERVED/undefined — no source, doc, or usage defines it (the sole reference is a dangling `matchesSovereigntyMode` in an informative snippet), and inventing one (where does `bidding` sit?) is design-by-spec. A future `Updates:` may define an ordering — widening match-acceptance is compatible; narrowing is not. "Minimum" in the name recorded as aspirational, not semantic. Closes RFC-0005 OD-7 by reference (0005 §2.6 cites here).

## Determined (batch)

- **D4 (§6.3) — KV key grammar → DID-keyed, recorded historically.** The shipped store is DID-keyed; with D2 retiring the KV contract this is recorded as the v-current shape in the now-informative §5, not a live normative choice.
- **D5 (§6.4) — trust-boundary validation → MUST.** The presence-path receiver MUST validate each `capabilities[]` entry against the D1 grammar, bound `sovereignty` to its enum, and range-check `load`/`maxConcurrent` before folding into the registry (security-first; the §9.1 runtime-guard gap becomes a named conformance defect). Applies to the presence wire (D2), superseding the F-11-scoped framing.
- **Reserved tags (§4.3/§9.4):** reservation enforcement rides the same D5 validation gate (reject a reserved tag from an unauthorized advertiser) — named defect until enforced.
- **D1 sub-decision — the converged MATCH rule → segment-prefix.** A requirement matches an advertisement iff the requirement's segments are a prefix of the advertisement's segments (`code-review` matches `code-review.typescript`; equal ids trivially match). Grounded twice: the live cortex matcher is prefix-shaped, and the RATIFIED subject tree routes `tasks.code-review.>` inclusively — the wire's routing is prefix-shaped by construction. myelin's exact-membership `caps.includes(tag)` = named conformance defect. Converged segment grammar: each dot-separated segment IS a `capability-tag` (the task-side kebab rule, 2–64, no `_`, no trailing/consecutive `-`) — so the single-segment form is literally backward-compatible.
- **§9 findings:** re-frame per D1/D2/D5 resolutions (unwritten signed perimeter → the 0004 envelope IS the perimeter on the canonical wire; two-parallel-wires → resolved by retirement; self-registration/economics/unbounded-content stay recorded findings with SHOULD-bounds).
