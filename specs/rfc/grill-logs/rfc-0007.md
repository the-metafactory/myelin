## The two Andreas calls (2026-07-15)

### D2 — Token spelling → **snake_case canonical**
Three live spellings existed (myelin kebab `NakReason`, cortex snake reader+emitter, arch-doc snake MUST). **snake wins on payload-plane consistency**: reason tokens are PAYLOAD fields and the entire ratified payload plane is snake (`identity_mismatch` RFC-0006, `signed_by`/`correlation_id` RFC-0003, `not_now` admission.md §7). Kebab remains canonical ONLY for subject segments (RFC-0002/0001 kebab-strict). Set: `cant_do | wont_do | not_now | compliance_block`. **Migration**: myelin's single 4-token emitter (`src/lifecycle/types.ts:6` NakReason) flips at flag-day R — the cheapest possible surface, under our control; cortex's live read path (`failed-dispatch.ts:78-88`), `dispatch-events.ts` emitter, arch-doc MUST, and admission.md are already right. Dual-accept on receive during the window (kebab aliases normalize → snake). The docket's kebab lean is REJECTED: it would migrate more surface, make the payload plane inconsistent, and diverge from the just-ratified RFC-0006 token precedent. Stress note honored: no new payload version field in 0007 (wire versioning is BCP-0001's; don't re-own).

### D8 + D3 — Boundary ownership → **pin the layered carve in 0007; amend the RFC-0010 charter in the same commit**
The carve itself is the STRESS-CORRECTED layering (D1): **RFC-0007 owns the closed NAK reason VALUE SET (4 tokens) + delivery dispositions; RFC-0010 owns the refusal OBJECT `{kind, detail, retry_after_ms}` — its field grammar, kind registry, transient-vs-permanent rule, carriage (incl. on `nak(retry_after_ms)`), AND the object↔token seam-consistency rule; RFC-0006 owns membership-boundary rejects.** One reason can be a 0007 token *wearing* a 0010 object — a layering, not a trichotomy (`not_now` is the proof: transport token driving backoff AND refusal kind carrying retry_after_ms, simultaneously, no contradiction). The docket's "delivery-outcome vs refusal-cause" single axis is REJECTED (fails to partition `not_now`; would evict `wont_do`/`compliance_block` from §3.1; pulls `nak(retry_after_ms)` carriage into 0007). Governance: RFC-0010's charter line "the boundary is settled at 0010's grill" is AMENDED in the same commit ("boundary ratified at RFC-0007 §3; 0010 designs the far-side object grammar + owns the seam-consistency rule") — the charter is a Chartered draft, not ratified, so amending is free under the living-spec model and no two-owners contradiction survives. D3's proposed 0007-owned co-carriage MUST is NOT taken (it legislated over 0010's ceded carriage); the seam-consistency requirement is RECORDED IN THE 0010 CHARTER now, designed at 0010's grill.
