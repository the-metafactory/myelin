## Layer 0 — stance (RATIFIED, Andreas 2026-07-13)

- **L0a — Default bias = TIGHTEN AT THE CUT.** Where live implementation and best crypto practice
  diverge (malleable signature encodings, no domain separation, loose canonicalization corners),
  the spec defaults to the STRICT form, effective at flag-day R (hard cut regenerates every
  signature, so strictness is nearly free now, prohibitive later). Each tightening still surfaces
  individually in the grill; this sets the presumption. NEVER flip a live posture as part of this —
  spec-level rules only.
- **L0b — Conformance target = INDEPENDENT-IMPLEMENTATION GRADE.** A second implementation
  (Go/Python/Rust, no access to the TS source) MUST be able to verify byte-for-byte from spec +
  vectors alone. Consequences: the JCS profile is pinned to the byte (number serialization, string
  escapes, key ordering, absent-vs-null); every corner gets a conformance vector; "read the TS
  source" is never part of the contract.
