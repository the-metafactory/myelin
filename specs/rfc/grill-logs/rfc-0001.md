## Layer 1 — Lexical (D1–D6) — RATIFIED
- **Segment grammar:** `segment = lower *(lower / DIGIT / "-")` — leading letter, lowercase-only, **no `_`**, **no trailing `-`**, **no `--`** (kebab-strict). 1–63 chars/segment + whole-DID msi ceiling.
- **Comparison:** byte-exact, no normalization (case-fold/NFC/trim). RFC-0001 cites RFC-0004 for the signing-invariant, does not duplicate it.
- **Consequence accepted:** kebab-strict retracts the draft's "a stack-slug MAY end in a hyphen" (§3.2) and tightens `STACK_SEGMENT_REGEX`; 0 live slugs violate. Makes subject encoding injective BY GRAMMAR; retires the unsound first-hyphen decoder (review-consumer.ts).
- **Deferred:** federated-subject length blow-up (agent DID repeating {p}.{s}) → RFC-0002's short-form call; RFC-0001 only sets length caps.
- Editorial fixes folded in: D1 demo→negative reject vector; D4 justification corrected (leading-letter right, "subject-embeddability" reason wrong for assistant class).

## Layer 2 — Taxonomy (D7–D13) — RATIFIED
- **Class model = TWO PLANES, 6 tags.**
  - KEYED (hold Ed25519 key, verified, in `signed_by[]`): `principal`, `stack`, `agent`, `hub`.
  - SELF-ASSERTED (no key, in `originator` only): `surface` (external trust-anchors: github/discord/slack), `system` (internal non-keyed originators: reflex, signal-tap).
  - **NORMATIVE invariant:** a verifier MUST NOT resolve a self-asserted DID in the keyed registry (kills the `unknown_agent` bug class).
- **Corrections captured:** `did:mf:public` → `principal.public` (real keyed principal, NOT system). `wallet` (RFC-0009) → a role over any DID, no tag, name reserved in the §7 registry.
- **Hub-id == network-id** — one hub per network; `did:mf:hub.metafactory`; unique by construction (network_id is the registry PK). Multi-hub = future extension behind a real hub roster (none exists today).
- **D8 tag governance:** closed grammar + §7 tag registry, fail-closed on unregistered tags; unknown-tag REJECTION vector required.
- **D11 IdentityType:** widen enum to the 6-tag set; fix live bug — resolved peer principal stamped `type:'agent'` → `type:'principal'` (identity-registry.ts:362); BCP-0001 accept-first sequence.
- **D13 codec:** expose full `parse/render/encode/decode` as NORMATIVE + vector-bound; ~15 cortex splice sites collapse onto the vendored myelin codec; `decodeDidSegment` normative (injective under dot-form + kebab-strict).

## Layer 3 — DID-method spec (D14–D17) — RATIFIED

Self-asserted (surface/system) → explicitly NON-RESOLVABLE.
- **D15 wire form:** bare DID at every wire position (no path/query/fragment); fragments only as verificationMethod ids inside a resolved document; reject-vectors for `#`/`/`/`?`.
